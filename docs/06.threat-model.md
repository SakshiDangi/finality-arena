# Threat Model

## Overview

Finality Arena is designed to protect the prediction lifecycle against common integrity and security threats. The protocol combines cryptographic verification, replay protection, deterministic processing, and secure storage to ensure that predictions remain authentic and tamper-resistant.

---

# Security Objectives

The protocol aims to provide:

* Prediction integrity
* User authenticity
* Replay protection
* Deterministic settlement
* Transparent verification
* Auditable results

---

# Threats and Mitigations

| Threat                | Description                                                            | Mitigation                                                                                               |
| --------------------- | ---------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| Prediction Tampering  | An attacker modifies a prediction after submission.                    | Cryptographic hashing and digital signatures detect any modification.                                    |
| Forged Predictions    | An attacker submits predictions pretending to be another user.         | Wallet-based authentication and signature verification ensure authenticity.                              |
| Replay Attack         | Previously submitted predictions are resent to gain unfair advantages. | Replay protection validates unique requests and rejects duplicates.                                      |
| Invalid Settlement    | Incorrect match results are used during settlement.                    | Settlement is executed using verified official match outcomes through a deterministic engine.            |
| Duplicate Predictions | Multiple identical submissions are accepted.                           | Unique prediction identifiers and replay checks prevent duplicates.                                      |
| Data Corruption       | Stored prediction data becomes corrupted.                              | Digest verification detects integrity violations before processing.                                      |
| Unauthorized Access   | Untrusted users attempt administrative actions.                        | Administrative APIs are isolated and can be protected with authentication and role-based access control. |

---

# Trust Assumptions

Finality Arena assumes:

* Users control their own wallets.
* The Finality verification service behaves correctly.
* Official match results are accurate.
* Server infrastructure is securely managed.

---

# Out of Scope

The current prototype does not address:

* Wallet key compromise
* Network-level denial-of-service attacks
* Smart contract exploits
* Oracle manipulation
* Infrastructure compromise

These concerns can be addressed in future production deployments.

---

# Security Principles

The protocol follows these principles:

* Verify before storing.
* Reject invalid or duplicate submissions.
* Use deterministic processing.
* Minimize trust in centralized components.
* Maintain complete auditability.

---

# Conclusion

By combining wallet authentication, cryptographic verification, replay protection, and deterministic settlement, Finality Arena reduces common attack vectors while ensuring every accepted prediction remains transparent, verifiable, and trustworthy throughout its lifecycle.
